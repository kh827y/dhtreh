# Аудит: merchant-portal /wallet

Ниже — найденные проблемы по странице кошелька (merchant-portal). Отсортировано по убыванию важности.

## P1 — Высокий приоритет

1. **Страница /wallet — заглушка без функционала и данных.**
   - **Риск:** мерчант не может видеть балансы/историю по кошелькам, нет операционного контроля и проверки корректности начислений/списаний. Для продакшена это блокирует использование раздела «Wallet».
   - **Где видно:** `merchant-portal/app/wallet/page.tsx` содержит только статический заголовок и текст «Раздел-заглушка», нет запросов к API, таблиц балансов/операций, фильтров и т.п.

## P2 — Средний приоритет

1. **Нет связи с ledger и отсутствует доступ к деталям операций кошелька.**
   - **Риск:** даже при наличии данных в бэкенде нет интерфейса для просмотра ledger‑записей/движения баллов; это усложняет разбор инцидентов, аудит начислений/списаний и поддержку клиентов.
   - **Где видно:** в API есть endpoint `GET /portal/ledger` (portal controller), но в `merchant-portal/app/wallet/page.tsx` отсутствует какая‑либо интеграция с ledger или отображение истории операций.
