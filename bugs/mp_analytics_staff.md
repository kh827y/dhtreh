# Аудит: merchant-portal / Аналитика персонала

Ниже перечислены найденные проблемы по странице `merchant-portal/app/analytics/staff/page.tsx` и связке с backend (операции/мотивация). Недоработки отсортированы по убыванию важности.

## P1 — Высокая

1. **Не учитываются операции с нулевым чеком (100% списание/бесплатная покупка)**
   - **Где**: `api/src/analytics/analytics.service.ts`, метод `getStaffMetrics()` — фильтры `r."total" > 0` в `staff_receipts` и `base_receipts`.
   - **Суть**: все чеки с `total = 0` исключаются из расчётов по персоналу (кол-во чеков, выручка, списания, новые клиенты). При 100% оплате баллами сотрудник фактически сделал продажу/обслужил клиента, но статистика покажет ноль.
   - **Риск**: данные по KPI персонала занижаются, руководитель видит неверную эффективность сотрудников и нагрузку по списаниям.

## P2 — Средняя

2. **Очки мотивации показываются даже когда программа мотивации выключена**
   - **Где**:
     - UI: `merchant-portal/app/analytics/staff/page.tsx` (колонка “Очки”, карточка “Лучший сотрудник”).
     - Backend: `api/src/staff-motivation/staff-motivation.engine.ts` (`recordPurchase()` возвращает 0, если `settings.enabled = false`).
   - **Суть**: при выключенной мотивации `staffMotivationEntry` не пишутся, из‑за чего `performanceScore` всегда равен 0, но интерфейс не сообщает, что программа отключена. Пользователь видит “Лучший сотрудник” с 0 очков и может считать аналитику сломанной.
   - **Риск**: недоверие к аналитике и неправильные управленческие решения (особенно когда мотивация отключена сознательно).

3. **Пороговая раскраска “Очков” не соответствует реальной шкале мотивации**
   - **Где**: `merchant-portal/app/analytics/staff/page.tsx` — цветовой порог `>= 90`/`>= 80`.
   - **Суть**: `performanceScore` — это суммарные мотивационные баллы (по умолчанию 10/30 баллов за покупку), а не нормализованный KPI 0–100. При типичных объёмах продаж значение быстро превышает 100, а при малых — почти всегда ниже 80. Цветовая шкала в итоге не отражает реальную эффективность.
   - **Риск**: вводит в заблуждение при сравнении сотрудников и оценке эффективности.
